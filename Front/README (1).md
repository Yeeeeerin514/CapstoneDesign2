# 알바지킴이 (AlbaJikimi)

> 아르바이트생의 노동 권리를 보호하고 임금체불 등 위법 사항을 해결하기 위한 종합 노동 권리 보호 앱

AI와 Wi-Fi BSSID 자동화 기술로 알바생의 노동을 지킵니다.

---

## 기술 스택

| 영역 | 선택 |
|---|---|
| 프레임워크 | **React Native (Expo SDK 51)** |
| 라우팅 | **Expo Router v3** |
| 언어 | **TypeScript strict** |
| 스타일링 | **NativeWind v4** |
| 상태관리 | **Zustand** |
| HTTP | **Axios** |
| AI | **Anthropic Claude API** |
| BSSID | **react-native-wifi-reborn** (Android 전용) |
| 아키텍처 | **Feature-Sliced Design (FSD)** |

## 빠른 시작

### 1. 의존성 설치
```bash
npm install
```

### 2. 환경변수 설정
```bash
cp .env.example .env.local
```
`.env.local`에 아래 값을 채웁니다:
- `EXPO_PUBLIC_API_URL` — 백엔드 서버 주소
- `EXPO_PUBLIC_ANTHROPIC_KEY` — Anthropic API 키

### 3. 개발 서버 실행
```bash
npx expo start
```

### 4. Android 에뮬레이터 실행
```bash
npx expo start --android
```

### 5. BSSID 기능 테스트 (실제 기기 필요)
```bash
eas build --platform android --profile development
```

### 기능 1. 홈 화면 — 알바 공고 검증

#### 개요
사용자가 알바몬/알바천국 등의 공고문 URL 또는 캡처 이미지를 입력하면, Claude AI가 사업장 정보를 추출하고 공공데이터와 대조하여 위법 사항 및 의심 요소를 찾아줍니다.

#### 화면 흐름
```
홈 (URL 입력 / 이미지 업로드)
  → 분석 중 (로딩)
    → 분석 결과 (위법 요소 리스트)
      → [관심업장으로 등록하기] 버튼
```

#### 기능 요구사항
- 알바몬/알바천국 공고 URL 입력란 + [분석시작] 버튼
- 이미지 캡처 업로드 대안 제공 (카메라/갤러리)
- 분석 중 단계별 진행 표시 (사업장 정보 추출 → 공공데이터 대조 → 위법 요소 검토)
- 분석 결과 화면:
  - 위험도 배지: 🔴 위법 / 🟡 주의 / 🔵 참고
  - 각 이슈별 카드 (제목, 설명, 법적 근거, 권고사항)
  - 하단 CTA: [관심업장으로 등록하기] / [다시 분석]
- 홈 화면 하단: 최근 분석 공고 목록 2–3건 + 체불 해결 후기 미리보기

#### Claude API 프롬프트 항목
최저임금 준수 여부 (2026년 10,030원), 주휴수당 명시 여부, 연장수당 규정, 4대보험 기재 여부, 근무시간 명확성, 사업장 실존 여부

### 기능 2. 관심업장 & 근로계약서 분석

#### 탭: 관심업장

#### 화면 흐름 (계약서)
```
[2-1-1] 관심업장 목록
  → [계약서] 버튼 클릭
    → [2-1-2] 계약서 업로드 폼
      → 파일/카메라 선택
        → [2-1-3] 업로드 완료 / 분석 중
          → [2-1-4] 계약서 분석 결과
            → [사업장 등록하기] 버튼
              → [2-1-5] 근무 스케줄 입력 (협의인 경우)
                → [2-2-2] BSSID 등록
                  → [2-2-3] 사업장 등록 완료
```

#### 화면 흐름 (업장 등록)
```
[2-2-1] 관심업장 목록 → [업장등록] 버튼
  → [2-2-2] BSSID 등록
    → [2-2-3] 사업장 등록 완료
```

#### 관심업장 목록 (2-1-1)
- 업장 카드: 업장명, 등록일, 현재 상태 배지
  - 상태: 계약서 미업로드 / 계약서 분석 완료 / BSSID 미등록 / 등록 완료
- 각 카드에 [계약서] [업장등록] 인라인 버튼

#### 계약서 업로드 (2-1-2)
- 드래그/클릭 업로드 영역 (JPG, PNG, PDF / 최대 10MB)
- [카메라로 촬영] [파일 선택] 버튼
- 필수 확인 사항 + 촬영 팁 안내
- 개인정보 보호 안내 배너

#### 계약서 분석 결과 (2-1-4)
- 분석 요약 카드: 사업장명, 계약 기간, 시급, 예상 월급
- 위험도 배지: 🔴 주의 필요 N건 / ✅ 이상 없음
- 탭 구조: [발견 이슈 (N)] [계약 내용] [법률 가이드]
- 발견 이슈 탭:
  - 🔴 위법: 빨간 카드
  - 🟡 주의: 노란 카드
  - 🔵 참고: 파란 카드
  - 각 카드: 제목, 설명, 법적 근거, 권고사항
- 이상 없음 시: 초록 배지 + 법률 가이드 접힘 처리
- CTA: [사업장 등록하기] [계약서 다시 업로드]

#### 근무 스케줄 입력 (2-1-5) — 공고에 '협의'인 경우
- 진입 시점: [사업장 등록하기] 클릭 직후 별도 전체 화면으로 진입 (모달 아님)
- 안내 배너: "이 정보는 수당 계산의 기준이 됩니다"
- 근무 요일 선택: [월][화][수][목][금][토][일] 토글
- 출근 시간 / 퇴근 시간 / 휴게 시간 피커
- 하단 계산 미리보기: 실 근무시간, 주휴수당 발생 여부, 예상 월급

#### BSSID 등록 (2-2-2)
- BSSID 개념 설명 카드 (고유 식별자, 30분 단위 확인, 법적 효력)
- 주변 Wi-Fi 네트워크 스캔 목록 (신호 강도 순, [추천] 뱃지)
- [↻ 다시 스캔] 버튼
- 항목 선택 후 [Wi-Fi 선택해주세요] → [등록] 버튼 활성화

#### 사업장 등록 완료 (2-2-3)
- 성공 체크 애니메이션
- 등록된 BSSID 정보 카드 (네트워크명, BSSID, 확인 주기 30분, 활성화 상태)
- 자동 근무 기록 방식 설명 (출근 감지 → 근무 중 모니터링 → 퇴근 감지)
- 주의사항
- CTA: [근무 대시보드로 이동] [홈으로]

### 기능 3. 근무기록 탭 & 근무 대시보드

#### 탭: 근무기록

#### 화면 흐름
```
[3-1] 근무 업장 리스트
  → [근무기록] 버튼 클릭
    → [3-2] 근무 대시보드 (업장별)
```

#### 근무 업장 리스트 (3-1)
- 섹션 제목: "근무 대시보드 — 자동으로 기록된 근무 시간과 수당을 확인하세요"
- 등록 완료된 업장 카드 목록:
  - 업장명, 근무 요일/시간, 시급
  - 이번 달 누적 시간 & 예상 급여
  - [근무기록 →] 버튼
- BSSID 미등록 업장은 [등록 완료하기] 버튼과 함께 경고 카드로 표시

#### 근무 대시보드 (3-2)
- 현재 상태 배너:
  - 근무 중: 실시간 타이머, 출근 시각, [● LIVE] 뱃지
  - 오프라인: 00:00 / 오늘 근무 종료
- 실시간 알림 뱃지:
  - 주 40시간 도달 임박 시 경고
  - 주휴수당 발생 조건 충족 시 안내
- 통계 카드 3개: 주간 근무시간 / 주간 수당 / 이번 달 출근 횟수
- 최근 근무 기록 리스트:
  - 날짜, 출퇴근 시각, 근무 시간, 예상 수당
  - 연장근무 / 야간수당 발생 시 색상 강조
- 하단 CTA: [임금체불 신고하기] 버튼

#### 수당 자동 계산 로직 (shared/lib/wage-calculator.ts)
- 기본급: 실 근무시간 × 시급
- 연장수당: 8시간 초과분 × 시급 × 1.5
- 야간수당: 22:00–06:00 근무 시간 × 시급 × 0.5 추가
- 주휴수당: 주 15시간 이상 근무 시 1일치 시급 자동 추가
- 최저임금 미달 시 경고 배지 표시 (2026년 기준: 10,030원/시)

### 기능 4. 출퇴근 알림 및 연장근무 처리

#### BSSID 기반 자동 출퇴근 감지 로직
```
1. 출근 감지: 지오펜싱 100m 진입 + 등록 BSSID Wi-Fi 연결 → 출근 알림 전송
2. 근무 중 모니터링: 30분마다 BSSID 연결 확인 → 실시간 근무 증명 생성
3. 퇴근 감지: 계약서 기준 퇴근 시각 도달 → 퇴근 알림 전송
4. Wi-Fi 연결 해제 + 지오펜싱 이탈 → 자동 퇴근 처리
```

#### 화면 흐름
```
[4-1] 외부 배너 알림 (출근) → 탭 →
  [4-2] 앱 내 출근 팝업 모달
    → [출근하기] → 근무 기록 시작
    → [나중에] → 무시

[4-3] 외부 배너 알림 (퇴근) → 탭 →
  [4-4] 앱 내 퇴근 팝업 모달
    → [퇴근하기] → [4-6] 퇴근 처리 화면 → [4-7] 퇴근 완료
    → [연장근무] → [4-5] 연장근무 설정 → [4-6] → [4-7]
    → [나중에] → 무시
```

#### 출근 알림 (4-1 외부 배너 / 4-2 앱 내 팝업)
- 외부 배너: "출근하시겠습니까? / 업장명 · 오늘 HH:MM 출근 예정 / [출근하기] [나중에]"
- 앱 내 팝업 (파란 헤더):
  - 예정 출근 시간, 현재 시각, 근무지
  - [나중에] [출근하기] 버튼

#### 퇴근 알림 (4-3 외부 배너 / 4-4 앱 내 팝업)
- 외부 배너: "퇴근 시간이 됐습니다 / [퇴근하기] [연장근무] [나중에]"
- 앱 내 팝업 (주황 헤더):
  - 총 근무 시간, 현재 시각
  - [퇴근하기] (주황) / [연장근무] (흰색) / [나중에] (회색) 버튼

#### 연장근무 설정 (4-5)
- 원래 퇴근 시간, 근무지 표시
- 연장 시간 선택: [+/-] 스피너 (시간/분) + 빠른 선택 버튼 [30분] [1시간] [1시간30분] [2시간]
- 새 퇴근 예정 시간 실시간 계산 표시
- [연장근무 확정] 버튼

#### 퇴근 처리 (4-6)
- 3단계 검증 표시:
  1. Wi-Fi 영역 확인 (연결됨/미연결)
  2. BSSID 일치 확인 (일치/불일치)
  3. 근무 시간 확인 (출근 시각, 퇴근 예정, 현재 시각)
- [→ 퇴근하기] 버튼 (주황)

#### 퇴근 완료 (4-7)
- 주황 성공 배너 + 오늘 근무 요약 카드 (근무지, 출퇴근 시각, 총 근무시간, 오늘 급여 예상)
- 근무 대시보드 미니 카드 (이번 주/달 통계)
- 최근 근무 기록 리스트

### 기능 5. 신고 탭 — 진정서 제출 플로우

#### 탭: 신고

#### UX 결정: GNB 독립 탭
신고 기능은 증거 업로드, 연대 매칭, 진정서 생성 등 여러 단계를 가진 독립 플로우이므로 GNB 탭으로 분리. 신고 내역이 없는 유저는 빈 화면(온보딩 안내) 표시.

#### 화면 흐름
```
[5-0] 신고 탭 (신고 없음: 빈 화면 + [신고 시작하기])
  → [5-1] 업장 선택
    → [5-2] 증거 업로드 + 연대하기 선택
      → [5-3] 연대 매칭/현황 (총대 우대 안내 포함)
        → [6-1] 진정서 초안 뷰어 (PDF)
          → [6-2] 노동부 제출 URL 안내
```

#### 신고 없음 화면 (5-0)
- 일러스트 + "아직 신고 내역이 없어요"
- 임금체불 케이스 안내 리스트 (퇴직 후 14일 미지급, 최저임금 이하 등)
- [신고 시작하기 →] 버튼

#### 업장 선택 (5-1)
- 근무 기록이 있는 업장 목록
- 각 카드: 업장명, 근무 기간, 추정 체불액
- [선택하기] 버튼

#### 증거 업로드 + 연대 (5-2)
- 업장 정보 + 추정 체불액 (BSSID 기록 기반 자동 계산)
- 자동 수집 증거: BSSID 근무 기록 ✅ / 근로계약서 ✅ / 통장 내역 ⬜ / 급여명세서 ⬜
- [+ 추가 증거 업로드] 버튼
- 연대하기 섹션:
  - "현재 같은 업장 피해자 N명 확인됨 🔔"
  - [연대하여 함께 신고하기 →] [개인으로 신고하기]

#### 연대 현황 (5-3)
- 연대 참가자 목록: 이름(익명), 체불액
- 총 체불액 합산
- 총대(대표자) 우대 안내 카드:
  - 체불액 최고자 자동 지정
  - 혜택: 진정서 대표 서명권, 멘토 매칭 최우선 노출, 수익 배분 가중치 +20%
- [진정서 초안 생성하기 →] 버튼

#### 진정서 초안 뷰어 (6-1)
- PDF 미리보기 (진정인, 피진정인, 체불 금액, 진정 이유, 증거 목록)
- [내용 수정하기] [PDF 다운로드] 버튼
- [노동부에 제출하기 →] CTA

#### 노동부 제출 안내 (6-2)
- 온라인: 고용노동부 민원마당 링크
- 방문: 가까운 지방고용노동청 찾기
- 우편: 관할 주소
- 첨부 서류 체크리스트

### 기능 6. 멘토-멘티 매칭 & 에스크로 결제

#### 개요
임금체불 신고 성공자(멘토)와 현재 진행자(멘티)를 증거 유사성/체불 성격 기반으로 AI 매칭합니다.

#### 비즈니스 로직
- 멘티가 조언을 구하며 소정의 조언료 결제 (에스크로)
- 사건 해결 시 조언료의 30–40%를 멘티에게 환급
- 멘토(선배): 성공한 사람만 등록 가능

#### 화면 흐름
```
MY탭 > [멘토 찾기]
  → [멘토 목록] (AI 추천, 유사도 순)
    → [멘토 프로필] (상세 정보)
      → [에스크로 결제] (조언료 + 환급 안내)
        → [매칭 완료] → 채팅/조언 화면
```

#### 멘토 추천 목록
- 나의 사례 요약 표시 (업종, 체불 유형, 금액)
- 추천 멘토 카드:
  - 유사도 % (막대 표시)
  - 유사 항목 체크리스트 (동일 업종, 동일 체불 유형, 해결 방법)
  - 소요 기간, 조언료, 해결 시 환급 금액
  - 평점 (별점)
- [멘토 선택하기 →] 버튼

#### 에스크로 결제
- 에스크로 개념 설명 카드 ("결제 금액이 즉시 멘토에게 전달되지 않음")
- 결제 금액 + 해결 시 환급 금액 강조 표시
- 결제 수단 선택 (카드, 카카오페이, 토스)
- 약관 동의 체크박스
- [결제하기] 버튼

### 기능 7. 총대(대표자) 우대 시스템

#### 개요
공동 대응(연대) 시 체불액이 가장 높은 사람을 자동으로 총대(대표자)로 지정합니다.

#### 총대 지정 조건
- 연대에 참여한 피해자 중 체불액 최고자 자동 지정
- 체불액 변동 시 실시간 재지정
- 추가 피해자 참여 시 재계산

#### 총대 혜택
| 혜택 구분 | 내용 |
|---|---|
| 즉시 혜택 | 진정서 대표 서명자 등록, 노동청 접수 최우선 처리 요청 가능 |
| 해결 후 혜택 | 멘토(선배) 등록 시 검색 최상단 노출 |
| 수익 혜택 | 멘티 매칭 수익 배분 시 +20% 가중치 |
| 배지 | 성공 후기 게시판 '총대 인증 배지' 자동 부여 |

#### 노출 위치
- 연대 현황 화면 (5-3): 총대 우대 안내 카드
- 총대 지정 시 인앱 배너 알림
- 총대 혜택 상세 모달

### 기능 8. 지역/유형별 임금체불 해결 후기 게시판

#### UX 결정: GNB 독립 탭 + 홈 화면 하단 미리보기 병행

GNB 탭으로 배치해 진입 장벽을 낮추고, 홈 화면 하단에도 최신 후기 2–3개를 노출해 서비스 신뢰도를 초기부터 전달합니다.

#### 작성 권한
- **읽기**: 누구나 가능
- **작성**: 임금체불 해결 성공자만 가능 (알바지킴이 내 해결 인증 필요)

#### 화면 흐름
```
후기 탭
  → [목록] (지역/유형 필터)
    → [상세 보기]
      → (해결 성공자만) [후기 작성하기] 플로팅 버튼
```

#### 후기 목록
- 필터: [지역 ▼] [체불 유형 ▼]
- 후기 카드: 업종 뱃지, 제목, 체불금액, 해결금액, 소요기간, 해결 방법, 좋아요/댓글 수
- 총 해결 건수 표시
- 하단 플로팅 버튼: [+ 후기 작성하기] (해결 성공자만 표시)

#### 후기 상세
- 업종/유형/지역 태그
- 작성자 익명 표시 + 해결 인증 배지 ✅
- 상황 설명, 해결 과정, 조언
- 댓글 섹션
